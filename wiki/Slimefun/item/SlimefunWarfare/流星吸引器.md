# MeteorAttractor 流星吸引器 工作原理详解
## 一、类定义与基础属性
### 1. 类继承关系
`MeteorAttractor` 继承自 `SimpleSlimefunItem<BlockUseHandler>`，是**Slimefun4插件自定义机器**，核心触发方式为**玩家右键方块**。

### 2. 核心静态属性
```java
private static final CoolDowns cooldowns = new CoolDowns(1000L * 60L *
    SlimefunWarfare.inst().getConfig().getInt("space.attractor-cooldown", 1)
);
```
- `CoolDowns`：全局冷却管理器，**绑定玩家UUID**，而非机器、方块或世界
- 冷却时长：配置文件读取 `space.attractor-cooldown`，默认1分钟（毫秒单位）
- 全局共享：所有流星吸引器共用同一个冷却实例

## 二、构造方法：物品注册
```java
public MeteorAttractor() {
    super(Categories.MACHINES, Items.METEOR_ATTRACTOR, RecipeType.ENHANCED_CRAFTING_TABLE, new ItemStack[]{
        Items.NDFEB_ALLOY_BLOCK, null, Items.NDFEB_ALLOY_BLOCK,
        Items.NDFEB_ALLOY_BLOCK, null, Items.NDFEB_ALLOY_BLOCK,
        Items.TERFENOL_D_BLOCK, Items.NDFEB_ALLOY_BLOCK, Items.TERFENOL_D_BLOCK
    });
}
```
- 分类：机器分类（`Categories.MACHINES`）
- 合成方式：高级合成台
- 配方：钕铁硼合金块 + 铽镝铁合金块

## 三、核心功能：drop() 方法（流星坠落执行）
### 1. 权限校验（边界/安全处理）
```java
if (!Slimefun.getProtectionManager().hasPermission(p, b, Interaction.BREAK_BLOCK)) return;
```
- **边界作用**：校验玩家对目标位置的破坏权限，兼容领地、地皮等保护插件
- 无权限直接终止逻辑，**防止跨领地/无权限生成陨石**，避免破坏玩家建筑

### 2. 坐标生成（边界处理）
```java
int radius = config.getInt("space.meteor-drop-radius", 100);
int x = ThreadLocalRandom.current().nextInt(b.getX() - radius, b.getX() + radius + 1);
int z = ThreadLocalRandom.current().nextInt(b.getZ() - radius, b.getZ() + radius + 1);
```
- **边界规则**：以右键的机器方块为中心，在`±半径`范围内随机生成X/Z坐标
- 硬边界限制：坐标不会超出配置的半径范围，**避免流星坠落到过远位置**

### 3. 世界高度匹配（边界/防卡死处理）
```java
world.getHighestBlockYAt(x, z)
```
- 自动获取对应X/Z坐标的**世界最高方块Y轴高度**
- **边界作用**：避免陨石生成在地下、虚空或建筑内部，保证陨石可被玩家获取

### 4. 方块可通过判断（边界/防异常处理）
```java
if (!landing.isPassable()) {
    landing = landing.getRelative(BlockFace.UP);
}
```
- 检测最高方块是否为实心/不可通过方块（如石头、建筑方块）
- **边界作用**：若目标位置被占用，自动向上移动一格，**保证陨石方块能成功放置**，不会生成失败

### 5. 爆炸与陨石生成
- 生成强度4的爆炸，模拟流星撞击
- 延迟2tick放置陨石（防止爆炸破坏陨石）
- 概率生成普通锇陨石/稀有铯陨石（按配置概率）
- 通过`BlockStorage`绑定Slimefun物品ID，让游戏识别自定义陨石

## 四、触发逻辑：getItemHandler() 方法
玩家**右键机器**时触发，核心流程：
### 1. 玩家绑定冷却判定（核心特性）
```java
if (cooldowns.checkAndReset(b.getPlayer().getUniqueId()))
```
- **冷却绑定对象**：**玩家UUID**，而非机器、方块或世界
- 逻辑：检查当前使用的**玩家**是否冷却完成，而非检查机器是否冷却

### 2. 随机延迟调度
- 读取配置：最小10分钟、最大30分钟，随机生成延迟时间
- 提示玩家流星坠落时间
- 延迟结束后执行`drop()`方法

### 3. 冷却中提示
- 玩家处于冷却时，发送红色提示，告知冷却时长

## 五、核心特性：多机器/多玩家机制
### 1. 多个流星吸引器 = 无额外效果
- 冷却系统**全局绑定玩家**，和机器数量无关
- 同一个玩家，无论右键多少个流星吸引器，都受同一个冷却限制
- **无法通过放置多台机器，实现同时召唤多颗流星**

### 2. 人越多，效率越高（核心特性）
- 冷却不绑定机器/世界，**只绑定玩家**
- 只要有**任意一台流星吸引器**，多个玩家可以各自独立使用
- 例：10个玩家共用1台机器，每人都能独立触发流星，互不干扰
- 结论：**机器数量不限制使用人数，只要有一台机器，全服玩家都能使用**

## 六、完整边界处理汇总
1. **权限边界**：校验玩家破坏权限，兼容保护插件，防止跨权限生成
2. **范围边界**：严格限制流星坠落半径，不超出配置范围
3. **地形边界**：自动匹配世界最高高度，不生成在地下/虚空
4. **方块边界**：检测方块是否可通过，自动向上偏移，保证生成成功
5. **安全边界**：延迟放置陨石，避免爆炸破坏自定义方块

## 七、完整工作流程
1. 玩家合成并放置流星吸引器
2. 玩家**右键机器**，系统检测**该玩家**的冷却状态
3. 冷却中：提示冷却时间，无法使用
4. 冷却完成：随机生成10~30分钟延迟，提示玩家
5. 延迟结束：在机器半径100格内，随机位置生成爆炸+陨石
6. 玩家冷却重置，1分钟后可再次使用
7. 多玩家共用一台机器：各自独立冷却，互不影响
8. 多台机器：无额外效果，仍受玩家冷却限制

## 八、关键配置项
| 配置键 | 作用 | 默认值 |
|--------|------|--------|
| `space.attractor-cooldown` | 玩家冷却时间（分钟） | 1 |
| `space.meteor-drop-radius` | 流星坠落半径 | 100 |
| `space.meteor-min-time` | 最小延迟时间 | 10 |
| `space.meteor-max-time` | 最大延迟时间 | 30 |
| `space.segganesson-chance` | 稀有陨石概率（%） | 0 |

## 九、高频问题官方解答
### 问题1：将机器放在世界边缘，是否能让流星坠落范围缩小到1/4，实现更密集的效果？
**答案：是，完全可以实现。**
- 流星生成逻辑：以机器为中心，在**X±半径、Z±半径**的正方形范围内随机选取坐标
- 若将机器放置在**世界边界（地图边缘）**，超出世界边界的坐标会被游戏自动限制在世界范围内
- 原本的**四向随机范围**会被边界截断为**单向/双向范围**，有效坠落面积缩小至原来的1/4甚至更小
- 最终效果：流星会**高度集中**在机器靠近世界内侧的区域，大幅提升陨石密集度

### 问题2：机器的最佳使用方式是冷却完成就点击，还是等流星落地再点击？
**答案：每1分钟冷却结束后立即点击，是最高效的玩法。**
- 核心机制：**冷却时间和流星延迟坠落时间是完全独立的两个机制**
- 冷却规则：1分钟冷却结束后，玩家就可以再次触发新的流星任务，无需等待上一颗流星落地
- 效率对比：
  - 低效：等10~30分钟流星落地后再点击 → 每30分钟最多触发1次
  - 高效：冷却1分钟到点就点击 → 每分钟都能触发一次新流星，单位时间召唤数量最大化
- 结论：**冷却完成立即点击，和上一颗流星是否落地无关**

### 问题3：多名玩家同时点击同一台流星吸引器是否允许？
**答案：完全允许，且是官方设计的正常机制。**
- 冷却绑定：系统只判定**玩家个人**的冷却状态，和机器数量、点击人数无关
- 多人机制：A玩家点击后，B/C/D玩家都可以正常点击，各自独立触发流星、独立计算冷却
- 实际效果：一台机器可供全服玩家同时使用，**人数越多，单位时间生成的流星总数越多**
- 无任何冲突：多人点击不会互相干扰，也不会导致机器失效

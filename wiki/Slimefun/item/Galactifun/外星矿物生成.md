## 1. 系统架构概览

Galactifun 的世界生成采用经典的 **继承 + 模板方法** 设计模式。核心类关系如下：

```
AlienWorld (抽象基类)
├── SimpleAlienWorld (提供噪声地形 + 逐方块生成框架)
│   ├── TheMoon (月球)
│   ├── Mars (火星)
│   ├── Venus (金星)
│   └── Io (木卫一)
├── FlatWorld (提供平坦分层地形框架)
│   ├── Europa (木卫二)
│   └── Enceladus (土卫二)
└── Titan (土卫六，直接继承 AlienWorld，完全自定义生成)
```

区块生成的完整流程：

```
生成基岩层 → generateNoise() → generateSurface() → getDefaultPopulators()
    │              │                │                      │
    │         填充方块           铺设表面             矿脉/火山/湖泊等
    │     (generateMaterial)    (getTop)           (BlockPopulator)
    │                                                     │
    └─────────────── addBlockMapping 映射 ←─────────────┘
              (原版方块 → Slimefun 掉落物)
```

**源文件位置参考：**
- `AlienWorld.java`：`src/.../api/worlds/AlienWorld.java`
- `SimpleAlienWorld.java`：`src/.../api/worlds/SimpleAlienWorld.java`
- `FlatWorld.java`：`src/.../api/worlds/FlatWorld.java`
- `OrePopulator.java`：`src/.../api/worlds/populators/OrePopulator.java`
- `BaseMats.java`：`src/.../base/BaseMats.java`

## 2. 核心生成机制详解

### 2.1 机制一：generateMaterial() — 逐方块基础生成

这是最基础的矿石生成方式。`SimpleAlienWorld` 在 `generateChunk()` 中遍历区块内每个坐标 (x, y, z)，调用子类重写的 `generateMaterial()` 方法来决定该位置放什么方块。

**核心代码** (`SimpleAlienWorld.java` 第 64-85 行)：

```java
for (int x = 0, realX = chunkX << 4; x < 16; x++, realX++) {
    for (int z = 0, realZ = chunkZ << 4; z < 16; z++, realZ++) {
        int height = getHeight(world, realX, realZ) - heightSub;
        int y = 1;
        while (y <= height) {
            chunk.setBlock(x, y++, z, generateMaterial(random, x, y, z, height));
        }
        generateMore(chunk, generator, random, realX, realZ, x, z, height);
    }
}
```

**方法签名：**
```java
protected abstract Material generateMaterial(Random random, int x, int y, int z, int top);
```

参数说明：
- `random` — 随机数生成器，每个区块独立
- `x, y, z` — 方块在区块内的坐标
- `top` — 该列的最高方块 y 坐标（地表高度）

**设计要点：**
- 该方法对每一个地下方块都会调用一次，因此返回值的概率分布直接决定了矿物在整层中的比例
- 高度 `y` 是相对于区块的，从 1 开始直到 `top`
- `generateMore()` 是可选的额外生成钩子，如火星用来生成洞穴

### 2.2 机制二：OrePopulator — 矿脉后处理生成

`OrePopulator` 是一个独立的 `BlockPopulator`，在区块基础方块铺设完成后运行。它的作用是生成**矿脉**——一种呈蛇形蜿蜒的矿石聚集形态。

**核心代码** (`OrePopulator.java` 第 54-82 行)：

```java
public void populate(WorldInfo worldInfo, Random random, int cx, int cz, LimitedRegion region) {
    int startX = region.getCenterChunkX() << 4;
    int startZ = region.getCenterChunkZ() << 4;

    for (int i = 0; i < this.attempts; i++) {
        if (random.nextInt(100) < this.chance) {
            int x = random.nextInt(16);
            int y = random.nextInt(this.maxy - this.miny) + this.miny;
            int z = random.nextInt(16);

            int length = 0;
            while (length < this.maxSize
                   && this.replaceable.contains(region.getType(startX + x, y, startZ + z))) {
                region.setType(startX + x, y, startZ + z, this.ore);

                if ((length < this.minSize) || (random.nextInt(100) < 50)) {
                    switch (random.nextInt(6)) {
                        case 0 -> x = Math.min(x + 1, 15);
                        case 1 -> y = Math.min(y + 1, this.maxy);
                        case 2 -> z = Math.min(z + 1, 15);
                        case 3 -> x = Math.max(x - 1, 0);
                        case 4 -> y = Math.max(y - 1, this.miny);
                        case 5 -> z = Math.max(z - 1, 0);
                    }
                    length++;
                }
            }
        }
    }
}
```

**参数说明：**

| 参数 | 类型 | 含义 |
|------|------|------|
| `attempts` | int | 每区块尝试生成矿脉的次数 |
| `chance` | int (0-100) | 每次尝试触发生成的概率 |
| `miny` | int | 矿脉生成的最低 Y 坐标 |
| `maxy` | int | 矿脉生成的最高 Y 坐标 |
| `minSize` | int | 矿脉最小长度（保证至少生成这么多格） |
| `maxSize` | int | 矿脉最大长度（超过后停止延伸） |
| `ore` | Material | 矿石方块类型 |
| `replaceable` | Material[] | 可被替换的原方块（只有这些方块才会被矿石替代） |

**矿脉生成算法解读：**

1. 对于每次 `attempt`，先以 `chance` 概率决定是否生成
2. 如果生成，随机选一个起始坐标 (x, y, z)
3. 从起始点开始，沿随机方向（上下左右前后共 6 个方向）蜿蜒延伸
4. 前 `minSize` 步一定会延伸，之后每步有 50% 概率停止
5. 每一步将目标方块替换为矿石，但仅当目标方块属于 `replaceable` 集合时才替换
6. 总长度不超过 `maxSize`

**唯一使用实例（土卫六 Titan）：**
```java
populators.add(new OrePopulator(
    1,                      // 每区块尝试 1 次
    50,                     // 50% 概率触发
    1,                      // y=1 开始
    40,                     // y=40 结束
    2,                      // 最小 2 格
    6,                      // 最大 6 格
    BaseMats.LASERITE_ORE,  // 激光矿石
    Material.STONE, Material.COAL_ORE  // 仅替换石头和煤矿
));
```

### 2.3 机制三：addBlockMapping() — 方块映射系统

这是 Galactifun 最巧妙的设计之一。由于 Slimefun 自定义方块使用 `BlockStorage` 系统存储数据，大量使用会严重影响性能。因此，Galactifun 的做法是：

**生成时使用原版 Minecraft 方块 → 玩家挖掘时掉落 Slimefun 自定义物品**

**核心代码** (`AlienWorld.java` 第 172-173 行)：

```java
public final void addBlockMapping(Material vanillaItem, SlimefunItemStack slimefunItem) {
    this.blockMappings.put(vanillaItem, slimefunItem);
}
```

**映射查询** (`AlienWorld.java` 第 177-179 行)：

```java
public SlimefunItemStack getMappedItem(Block b) {
    return this.blockMappings.get(b.getType());
}
```

**工作原理：**
1. 世界生成时，实际放置的是原版方块（如 `TERRACOTTA`）
2. 当玩家挖掘该方块时，插件拦截掉落物事件
3. 查询 `blockMappings` 映射表
4. 如果找到匹配项，将掉落物替换为对应的 Slimefun 物品（如"火星岩"）
5. 视觉上看起来玩家挖的是原版方块，但拿到的是自定义物品

**全部映射关系** (来自 `BaseMats.java` 的 `setup()` 方法，以及 `worldItem()` 辅助方法)：

```java
private static void worldItem(SlimefunItemStack item, AlienWorld... worlds) {
    ItemStack[] recipe = new ItemStack[worlds.length];
    for (int i = 0; i < worlds.length; i++) {
        AlienWorld world = worlds[i];
        recipe[i] = world.item();
        world.addBlockMapping(item.getType(), item);  // 关键：注册映射
    }
    new SlimefunItem(CoreItemGroup.BLOCKS, item, CoreRecipeType.WORLD_GEN, recipe)
        .register(Galactifun.instance());
}
```

### 2.4 机制四：自定义 BlockPopulator — 特殊生成

除了 `OrePopulator` 外，各星球还可以在 `getPopulators()` 方法中添加任意的自定义 `BlockPopulator`，实现特殊的生成逻辑。

**注册入口** (`AlienWorld.java` 第 122-129 行)：

```java
public List<BlockPopulator> getDefaultPopulators(World world) {
    List<BlockPopulator> list = new ArrayList<>(1);
    getPopulators(list);  // 子类实现此方法来添加自己的 Populator
    if (getSetting("generate-fallen-satellites", Boolean.class, true)) {
        list.add(new FallenSatellitePopulator(0.5));  // 所有星球都有机会生成坠毁卫星
    }
    return list;
}
```

常见的自定义 Populator：
- **VolcanoPopulator** — 生成火山结构（金星、木卫一使用）
- **LakePopulator** — 生成液体湖泊（金星生成岩浆湖，土卫六生成水湖）
- **陨星生成** — 火星独有的 `ANCIENT_DEBRIS` 生成逻辑
- **冰方块生成** — 火星独有的冰方块散布逻辑
- **树木生成** — 土卫六在特定生物群系生成菌类树木

## 3. 各星球矿物分布与挖掘指南

### 3.1 月球 (The Moon)

**基本参数：**
- 地形类型：SimpleAlienWorld（噪声地形）
- 平均高度：50，最大偏差：35（即高度范围约 15-85）
- 有陨石坑（使用默认 CraterSettings）
- 生物群系：恶地 (BADLANDS)
- 重力：月球重力（约地球 1/6）

**地下方块生成逻辑** (`TheMoon.java` 第 48-53 行)：

```java
protected Material generateMaterial(Random random, int x, int y, int z, int top) {
    if (random.nextFloat() > .02) {
        return Material.ANDESITE;       // 98% 安山岩 → 掉落 "月岩"
    } else {
        return Material.GOLD_ORE;       // 2% 金矿石 → 掉落 "月球芝士"
    }
}
```

**表面方块** (`getTop()`)：
- 最顶部 2 层：`LIGHT_GRAY_CONCRETE_POWDER` → 掉落 "月球尘埃"

**矿物列表：**

| 矿物 | 原版方块 | 生成概率 | 分布范围 | 映射掉落物 |
|------|---------|---------|---------|-----------|
| 月岩 | ANDESITE | 98% | 所有地下深度 | 月岩 (MOON_ROCK) |
| 月球芝士 | GOLD_ORE | 2% | 所有地下深度 | 月球芝士 (MOON_CHEESE) |
| 月球尘埃 | LIGHT_GRAY_CONCRETE_POWDER | — | 地表最顶层 2 层 | 月球尘埃 (MOON_DUST) |

**挖掘建议：**
- **月球芝士**：分布均匀，与深度无关，2% 概率。任意深度都可以挖到，建议在 y=20-50 的大面积矿道中挖掘
- **月岩**：基础填充方块，随便挖就能获得大量
- **月球尘埃**：只需铲除表面 2 层即可获得

**提示**：月球芝士用于合成聚变弹丸 (FUSION_PELLET)，是重要的后期材料，建议多储备。

### 3.2 火星 (Mars)

**基本参数：**
- 地形类型：SimpleAlienWorld（噪声地形）
- 平均高度：75，最大偏差：35（即高度范围约 40-110）
- 有洞穴生成（`generateMore` 方法）
- 生物群系：沙漠 (DESERT)

**地下方块生成逻辑** (`Mars.java` 第 49-56 行)：

```java
protected Material generateMaterial(Random random, int x, int y, int z, int top) {
    if (random.nextDouble() <= 0.1 && y <= 15) {
        return Material.IRON_ORE;       // y ≤ 15 时，10% 铁矿石
    }
    return Material.TERRACOTTA;         // 其余 90% 为陶土
}
```

**表面方块** (`getTop()`)：
- 最顶部 4 层：`RED_SAND` → 掉落 "火星尘埃"

**额外 Populator — 陨星生成** (`Mars.java` 第 87-102 行)：

```java
// 每个区块 1% 概率
if (random.nextInt(100) < 1) {
    int x = random.nextInt(16) + (region.getCenterChunkX() << 4);
    int z = random.nextInt(16) + (region.getCenterChunkZ() << 4);
    for (int y = 0; y < worldInfo.getMaxHeight(); y++) {
        if (region.getType(x, y, z).isAir()) {
            region.setType(x, y, z, Material.ANCIENT_DEBRIS);  // 陨星
            break;
        }
    }
}
```

**额外 Populator — 冰方块生成** (`Mars.java` 第 103-120 行)：

```java
// 每个区块 50% 概率
if (random.nextDouble() < 0.5) {
    int x = ...;
    int z = ...;
    int y = random.nextInt(30) + 1;  // y = 1-30
    GenUtils.generateSquare(region, new Location(null, x, y, z),
        Material.PACKED_ICE, random.nextInt(4) + 1);  // 1-5 大小的冰方块群
}
```

**矿物列表：**

| 矿物 | 原版方块 | 生成概率 | 分布范围 | 映射掉落物 |
|------|---------|---------|---------|-----------|
| 火星岩 | TERRACOTTA | 90% | 所有地下深度 | 火星岩 (MARS_ROCK) |
| 铁矿石 | IRON_ORE | y≤15 时 10% | 仅 y ≤ 15 | 原版铁矿石（无映射） |
| 火星尘埃 | RED_SAND | — | 地表最顶层 4 层 | 火星尘埃 (MARS_DUST) |
| 陨星 | ANCIENT_DEBRIS | 每区块 1% | 地表最高空气方块处 | 陨星 (FALLEN_METEOR) |
| 干冰 | PACKED_ICE | 每区块 50% | y=1-30，以方块群形式 | 干冰 (DRY_ICE) |

**挖掘建议：**
- **铁矿石**：**仅在 y ≤ 15 的浅层生成！** 最佳挖掘高度为 y=1-15，建议在极浅层水平挖掘
- **陨星**：极其稀有，每 100 个区块才出现 1 个。陨星会出现在区块内某个随机列的最高空气位置。建议探索地表寻找暴露在外的远古残骸方块
- **干冰**：y=1-30 范围内散布的冰方块群（1x1 到 5x5 不等），50% 的区块都有。可以在浅层地下挖掘寻找
- **火星岩**：基础填充方块，随处可得

**提示**：陨星 (FALLEN_METEOR) 是钨锭 (TUNGSTEN_INGOT) 的唯一来源之一（另一个来源是火星土著人的掉落物），是后期制作太空等级板和超负荷板的关键材料。

### 3.3 金星 (Venus)

**基本参数：**
- 地形类型：SimpleAlienWorld（噪声地形）
- 平均高度：100，最大偏差：50（即高度范围约 50-150）
- 无特殊额外生成（无 `generateMore`）
- 生物群系：玄武岩三角洲 (BASALT_DELTAS)
- **危险环境**：高温效应 10 级、93 倍大气压、有风暴和雷暴

**地下方块生成逻辑** (`Venus.java` 第 48-58 行)：

```java
protected Material generateMaterial(Random random, int x, int y, int z, int top) {
    if (y > 75) {
        return Material.BLACKSTONE;           // 高层: 黑石
    } else if (y > 9) {
        return Material.BASALT;               // 中层: 玄武岩
    } else if (y > 8) {
        return Material.YELLOW_TERRACOTTA;    // 过渡层 (y=9): 黄陶土 → "硫磺块"
    } else {
        return Material.BASALT;               // 底层: 玄武岩
    }
}
```

**Populator：**
- **VolcanoPopulator(117, MAGMA_BLOCK, LAVA)** — 在 y=117 处生成火山（岩浆块+熔岩）
- **LakePopulator(75, LAVA)** — 在 y=75 处生成岩浆湖

**矿物列表：**

| 矿物 | 原版方块 | 分布范围 | 映射掉落物 |
|------|---------|---------|-----------|
| 黑石 | BLACKSTONE | y > 75 | 无映射（原版物品） |
| 玄武岩 | BASALT | y ≤ 9 或 9 < y ≤ 75 | 无映射（原版物品） |
| 硫磺块 | YELLOW_TERRACOTTA | y = 9（仅一层！） | 硫磺块 (SULFUR_BLOCK) |
| 透气石 | MAGMA_BLOCK | 火山结构中 | 透气石 (VENTSTONE) |

**挖掘建议：**
- **硫磺块**：**仅存在于 y=9 这一层！** 由于只有一个方块厚度，实际挖掘中不容易直接找到。建议在 y=8-10 之间进行水平挖掘，或直接研磨金星的地表玄武岩来获取
- **透气石**：出现在火山结构中，由 `VolcanoPopulator` 生成。需要找到火山结构才能获取
- **警告**：金星环境极其恶劣（高温 10 级），必须穿高级太空服和携带氧气设备

**提示**：硫磺块可通过研磨石 (GRIND_STONE) 加工为硫酸盐 (SULFATE)。透气石 (VENTSTONE) 可在冶炼炉 (SMELTERY) 中烧制为火山锭 (VOLCANIC_INGOT)。

| 硫磺块 | YELLOW_TERRACOTTA | y ≤ 75 | 硫磺块 (SULFUR_BLOCK) |

**挖掘建议：**
- **硫磺块**：占据所有 y ≤ 75 的空间，**数量极多！** 与金星仅有一层硫磺不同，木卫一的整个地下几乎都是硫磺。在任意深度 y=1-75 挖掘即可大量获取
- 硫磺块在木卫一上取之不尽，是最容易获取硫磺的地点
- 透气石同样在火山结构中可获得（与金星相同的 VolcanoPopulator）

### 3.5 土卫六 (Titan)

**基本参数：**
- 地形类型：直接继承 `AlienWorld`，完全自定义生成（不使用 SimpleAlienWorld）
- 高度范围：约 55-85（通过 `55 + 30 * noise²` 计算）
- 有多生物群系系统（森林、冰冻森林、荒地、干冰平原、碳森林、冰冻碳森林）
- **危险环境**：寒冷 6 级、1.5 倍大气压、有天气

**地下方块生成逻辑** (`Titan.java` 第 66-81 行)：

```java

    if (random.nextBoolean()) {
        chunk.setBlock(x, y, z, Material.STONE);     // 50% 石头
    } else {

    }
}
```

**表面方块生成** (`Titan.java` 第 85-124 行)：

根据生物群系决定最顶部方块：
```java
Material material = height < 57 ? Material.BLUE_ICE : switch (biome) {
    case FOREST         -> random.nextBoolean() ? WARPED_NYLIUM : CRIMSON_NYLIUM;
    case FROZEN_FOREST  -> WARPED_NYLIUM;
    case WASTELAND      -> SAND;

    case CARBON_FOREST, FROZEN_CARBON_FOREST -> COAL_BLOCK;

```

**OrePopulator — 激光矿石矿脉** (`Titan.java` 第 190-199 行)：

```java
));
```

**Populator：**
- **树木生成**（两个独立的树木 Populator，分别在森林和冰冻森林生物群系生成诡异菌和绯红菌）
- **LakePopulator(58, WATER)** — 在 y=58 处生成水湖

**矿物列表：**

|------|---------|-------------|---------|-----------|
| 石头 | STONE | 50% | 所有地下 | 无映射（原版物品） |

## 6. 常见问题 (FAQ)

### Q1：为什么我挖到的是原版方块（如安山岩、陶土），而不是自定义矿物？

**A**：这是 Galactifun 的核心设计。世界生成时确实放置的是原版方块，但当你**挖掘**时，插件会通过 `addBlockMapping` 系统将掉落物替换为 Slimefun 自定义物品。如果你挖到的是安山岩却没掉落月岩，请检查：
1. Galactifun 插件是否正确加载
2. 你是否在正确的星球上
3. 对应的方块是否在映射表中（参见第 4 节的速查表）

注意：部分原版方块（如 `BLACKSTONE`、`BASALT`）没有映射，挖到的就是原版物品，这是正常的。

### Q2：为什么金星上很难挖到硫磺？

**A**：根据代码，金星的硫磺块 (`YELLOW_TERRACOTTA`) 仅生成在 **y=9 这一层**，厚度只有一个方块。而 y=10-75 全是玄武岩，y > 75 全是黑石。因此建议在 y=8-10 之间挖水平矿道。另外，硫磺块还可以在**木卫一**大量获取（y ≤ 75 的整个地下层都是硫磺），如果只需要硫磺，去木卫一效率更高。

### Q3：火星的陨星到底怎么找？

**A**：陨星 (`ANCIENT_DEBRIS`) 的生成逻辑是：每个区块有 1% 概率，在区块内随机选一个 (x, z) 列，然后从 y=0 向上扫描，在**第一个空气方块**处放置陨星。这意味着陨星会出现在地表上。建议在火星地表探索，寻找看起来像远古残骸的方块。由于生成概率极低（1%），可能需要探索大量区块才能找到。

### Q4：为什么木卫二的蓝冰挖不到甲烷冰？

**A**：因为木卫二 (`Europa`) 的蓝冰**没有注册 blockMapping**。虽然土卫六 (`Titan`) 的 `BLUE_ICE` 映射为甲烷冰，但这个映射是针对土卫六世界的。木卫二的蓝冰挖到的就是原版蓝冰。要获取甲烷冰，请前往土卫六。

### Q5：什么是 blockMapping？可以自己添加吗？

**A**：`blockMapping` 是 `AlienWorld` 的一个 `Map<Material, SlimefunItemStack>`，定义了原版方块到 Slimefun 物品的映射关系。在代码中通过 `world.addBlockMapping(Material, SlimefunItemStack)` 添加。如果你是开发者，可以在星球注册后调用此方法来添加新的映射。映射注册通常在 `BaseMats.setup()` 中通过 `worldItem()` 辅助方法完成。

### Q6：OrePopulator 的矿脉具体长什么样？

**A**：矿脉是一条从起始点开始向随机方向蜿蜒的矿石链。每一步随机选择 6 个方向之一（+x, -x, +y, -y, +z, -z）延伸一格。前 `minSize` 步一定会延伸，之后每步有 50% 概率停止。以土卫六的激光矿石为例（minSize=2, maxSize=6），矿脉看起来像一条 2-6 格长的弯曲"虫子"，嵌在石头或煤矿中。

### Q7：为什么有些星球有矿石生成，有些没有？

**A**：这取决于星球是否在 `getPopulators()` 中添加了 `OrePopulator`。在当前代码中，**只有土卫六使用了 OrePopulator**。其他星球的矿石主要通过 `generateMaterial()` 逐方块生成。这两种方式的区别在于：
- `generateMaterial()`：逐方块独立随机，产生均匀分布的散布矿石（如月球芝士的 2% 概率）
- `OrePopulator()`：生成矿脉，产生聚集的矿石（如激光矿石的 2-6 格矿脉）

### Q8：如何在代码中为某个星球添加新的矿石？

**A**：有两种方式：

**方式一：修改 `generateMaterial()`**
适用于均匀散布的矿石。在星球的 `generateMaterial()` 方法中添加新的概率分支即可。例如想在金星添加稀有矿石：
```java
if (random.nextDouble() < 0.01) {
    return Material.DIAMOND_ORE;  // 1% 概率生成钻石矿
}
```

**方式二：在 `getPopulators()` 中添加 OrePopulator**
适用于矿脉形式的矿石。在星球的 `getPopulators()` 方法中添加：
```java
populators.add(new OrePopulator(
    2,                      // 每区块尝试 2 次
    30,                     // 30% 概率
    1, 30,                  // y=1 到 y=30
    3, 8,                   // 矿脉 3-8 格
    Material.DIAMOND_ORE,   // 钻石矿
    Material.BASALT         // 替换玄武岩
));
```

无论哪种方式，如果需要让玩家挖到自定义 Slimefun 物品，还需要调用 `addBlockMapping()` 注册映射。

### Q9：各星球之间能否传送矿物？有什么限制？

**A**：矿物可以在星球之间自由携带和传送。但需要注意：
1. 前往不同星球需要使用对应的火箭（一阶/二阶/三阶/离子火箭）
2. 进入某些星球需要特定的太空装备（如金星需要耐高温装备，土卫六需要耐寒装备）
3. 在真空环境（如月球、地球轨道）需要携带氧气密封器或氧气再生器

### Q10：FlatWorld 类型的星球（木卫二、土卫二）能否添加矿石？

**A**：可以。虽然 `FlatWorld` 的 `generateChunk()` 是 `final` 的（不能重写），但你仍然可以在 `getPopulators()` 方法中添加 `OrePopulator` 或自定义的 `BlockPopulator` 来在平坦地形上后处理生成矿石。只需在星球类中重写 `getPopulators()` 并添加矿脉生成器即可。

# Minecraft 猫娘 AI 对话脚本 完整工作原理解析
这是一个**基于 Minecraft 日志监听 + 本地大模型 API** 的全自动 AI 对话脚本，核心功能：**实时读取 Minecraft 聊天日志 → 识别 @AI 消息 → 调用本地大模型生成猫娘风格回复 → 安全发送回游戏**，同时带有限流、上下文记忆、持久化保存等完整功能。

## 一、整体架构总览（核心工作流程）
```
启动程序 → 加载历史对话 → 监听游戏日志 → 识别玩家@AI消息
→ 整理上下文 → 调用本地大模型API → 生成猫娘回复
→ 限流防踢发送 → 保存对话历史 → 循环监听
```
**一句话总结**：它是游戏和 AI 之间的**全自动翻译+对话桥梁**，玩家在游戏里@AI，就能和设定好的猫娘对话。

## 三、完整数据流（从玩家说话到AI回复）
你可以直接对照这个流程理解代码：

1. **玩家在游戏输入**：`@AI 摸摸头`
2. 游戏把这句话写入 `latest.log` 日志
3. 脚本**0.5秒扫描一次**，发现新日志
4. 正则提取：玩家名=xxx，内容=`@AI 摸摸头`
5. 清洗内容：去掉@AI → `摸摸头`
6. 加载**最近50条历史对话**作为上下文
7. 拼接人设+历史+问题，发送给本地大模型
8. 模型返回猫娘回复：`(蹭蹭手心) 好舒服喵~【开心】`
9. 消息加入队列，**按限流规则**发送到游戏
10. 对话保存到本地文件，永久记忆
11. 玩家在游戏里看到蓝色的 AI 回复

## 四、核心亮点与设计思路
1. **增量读日志**：不占资源，只看最新消息
2. **上下文记忆**：50条对话，AI 能连贯聊天
3. **持久化保存**：重启不丢记忆
4. **严格限流**：防止被服务器踢出/封禁
5. **消息队列**：保证消息有序、稳定发送
6. **人设固定**：AI 始终是软萌猫娘，不跑偏
7. **全自动运行**：启动后无需任何操作
